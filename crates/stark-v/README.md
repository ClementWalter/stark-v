# stark-v

Library for building rust guest programs.

## Overview

The `stark-v` crate provides the support required to write guest programs that
run inside the stark-v zkVM. It handles program entry, stack initialization, and
program termination via custom RISC-V instructions.

## Features

- **Entry point macro** - `entry!(main)` sets up the guest entry point
- **Stack initialization** - Configures stack pointer at `0x0020_0400`
- **Custom terminate instruction** - Encodes exit codes in a RISC-V custom-0
  opcode
- **Panic handler** - Default panic handler that terminates with exit code 1

## Usage

```rust
#![no_std]
#![no_main]

stark_v::entry!(main);

fn main() {
    // Your guest code here
}
```

## Architecture

The runtime establishes the following execution flow:

1. `_start` (assembly) - Sets global pointer and stack pointer
2. `__starkv_start()` - Rust entry that calls `starkv_entry()`
3. `starkv_entry()` - Generated by `entry!` macro, calls user's main
4. `terminate::<0>()` - Issues custom RISC-V instruction to halt execution

## Memory Layout

| Region     | Address       |
| ---------- | ------------- |
| Stack top  | `0x0020_0400` |
| Text start | `0x0020_0800` |

## Custom Instructions

The crate defines a custom RISC-V instruction for program termination:

- **Opcode**: `custom-0` (`0x0b`)
- **funct3**: `0` (TERMINATE)
- **Immediate**: Exit code (8-bit)

## Conditional Compilation

The guest-specific code only compiles when targeting `zkvm`:

```rust
#[cfg(target_os = "zkvm")]
```

On other targets, stub implementations are provided to allow the workspace to
compile cleanly for testing and development.
